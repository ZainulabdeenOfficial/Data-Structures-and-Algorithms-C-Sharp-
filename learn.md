# Learn

A practical, repo-first learning path for **Data Structures & Algorithms in C#**.

> Goal: study the theory, then implement and practice problems in this repository.

## How to use this repository

1. **Start with the fundamentals** (read + code):
   - Read: `Theory.md`
   - Then open the corresponding folder(s) (for example: `Array/`) and study the implementations.

2. **Practice actively**
   - Re-type implementations (don’t just read).
   - Add 3–5 small test cases for every algorithm you study.
   - Write down time/space complexity for each solution.

3. **Track your progress**
   - Keep a simple checklist (copy the template below into your notes).

### Suggested progress checklist template

- [ ] Read the concept
- [ ] Implement from scratch
- [ ] Add test cases
- [ ] Analyze complexity (Big-O)
- [ ] Solve 2–5 practice problems
- [ ] Review after 7 days

---

## Recommended learning order

### 1) Big‑O & Fundamentals
- Time vs space complexity
- Big‑O, Big‑Ω, Big‑Θ
- Common patterns: nested loops, logarithms, recursion

**Do this in the repo:**
- Read `Theory.md`
- Add notes/examples to `Theory.md` as you learn (optional but encouraged)

### 2) Arrays
- Traversal, searching
- Two pointers
- Sliding window
- Prefix sums
- Sorting basics

**Do this in the repo:**
- Study `Array/`
- Add at least 1 new array problem solution and document it in `README.md`

### 3) Strings
- Frequency counting
- Pattern matching basics
- Two pointers on strings

### 4) Hashing (Dictionary/HashSet)
- Counting, grouping
- Deduplication
- Fast lookups

### 5) Linked Lists
- Traversal
- Reversal
- Fast/slow pointers

### 6) Stacks & Queues
- Monotonic stack
- Queue patterns

### 7) Trees
- DFS/BFS
- BST operations
- Traversals

### 8) Heaps / Priority Queue
- Top‑K
- Merge K sorted

### 9) Graphs
- BFS/DFS
- Shortest paths (intro)
- Union‑Find (DSU)

### 10) Recursion & Backtracking
- Subsets, permutations
- Constraint search

### 11) Dynamic Programming
- 1D DP
- 2D DP
- Knapsack patterns
